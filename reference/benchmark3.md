## 3. Structured Scenario State and Evaluation Protocol

### 核心聚焦问题

每个 benchmark instance 如何被形式化为结构化救援场景，以及该数据集如何支撑 UAV 森林火灾救援中的闭环决策评估。

### 写作要点

1. 将每个测试样例定义为 **structured rescue scenario state**，而不是普通图像样本或静态规划样本。该定义需要突出：一个 benchmark instance 同时包含感知输入、火险语义、风险区域、UAV 工程状态、任务约束和动态扰动，因此能够支撑从场景理解到任务执行的完整评估。

2. 使用结构化表示：

   ```text
   S = {O, F, R, U, C, E}
   ```

   其中：

   - **O: Aerial observations**  
     UAV 空中图像、多视角观测或仿真环境生成的场景感知输入。

   - **F: Fire-risk elements**  
     火点、烟雾、潜在扩散点、危险源等需要被识别或处置的火险元素。

   - **R: Risk-region abstraction**  
     由火险元素聚合得到的风险区域，用于支持任务优先级判断和空间任务分解。

   - **U: UAV operational states**  
     UAV 的位置、电量、载荷、通信状态、可用性和当前任务状态。

   - **C: Mission constraints**  
     续航、载荷、任务优先级、覆盖要求、安全边界、协同关系和可用资源等约束。

   - **E: Event-driven perturbations**  
     新火点、火势扩散、UAV 故障、电量不足、补给需求、通信退化等动态事件。

3. 强调 \(S\) 的作用：它不是单纯的数据字段集合，而是模型进行闭环救援决策的状态描述。不同场景可以通过改变 \(F,R,U,C,E\) 的组合来强调不同能力，例如风险识别、约束规划、事件重规划、资源管理和多 UAV 协同。

4. 将评估链条概括为：

   ```text
   O → F → R → P → P'
   ```

   其中：

   - **O → F**：从空中观测中识别火险元素；
   - **F → R**：将离散火险元素抽象为风险区域；
   - **R, U, C → P**：基于风险区域、UAV 状态和任务约束生成初始任务计划；
   - **P, E → P'**：在扰动事件发生后进行任务调整或重规划；
   - **P' checking**：检查最终计划是否满足覆盖、资源、时序、协同和可用性约束。

5. 将本文 benchmark 定位为 **benchmark dataset with an evaluation protocol**。也就是说，核心贡献不是构造一个单一排行榜分数，而是提供可复现的结构化场景、标准输入输出格式、执行日志记录方式，以及一组可解释评估指标。

6. 说明每个场景实例应包含的可发布内容：

   - 场景观测 \(O\)：UAV 视角图像或多视角观测；
   - 火险元素 \(F\)：火点、烟雾、潜在扩散点等目标标注；
   - 风险区域 \(R\)：由火险元素聚合得到的任务区域；
   - UAV 状态 \(U\)：位置、电量、载荷、通信状态和可用性；
   - 任务约束 \(C\)：续航、载荷、优先级、安全边界和协同限制；
   - 扰动事件 \(E\)：新火点、故障、补给需求、火势扩散和通信退化；。

7. 说明 benchmark 的标准输入输出。输入为结构化场景状态 \(S\)，模型输出为任务计划。输出可以包括风险区域识别结果、UAV 分配方案、飞行与处置动作序列、扰动后的重规划结果，以及最终约束检查结果。

8. 评估协议报告六个与救援任务直接相关的指标，而不强制将其合成为单一总分：

   - **Completion**  
     衡量火险目标或风险区域被成功处置的比例，反映任务覆盖程度。

   - **Response**  
     衡量从火险出现或被识别到有效处置的响应时间，反映任务时效性。

   - **Time**  
     衡量完整任务从开始到结束的总耗时，反映整体执行效率。

   - **Event stability**  
     衡量扰动事件发生后任务是否保持连续、是否需要频繁重规划或补给，反映动态场景下的稳定性。

   - **Resource pressure**  
     衡量 UAV 队伍的总能耗、单机最大能耗和补给需求，反映资源约束压力。

   - **Coordination**  
     衡量多 UAV 在任务动作数、飞行距离和执行时长上的负载均衡情况，反映协同执行质量。

9. 强调多指标评估的意义。该 benchmark 不要求所有模型能力被压缩为一个总分，而是通过多维指标解释模型差异。例如，较高的 completion 但较差的 response 表明模型覆盖充分但处置不够及时；较低的 resource pressure 但较差的 event stability 表明模型节约资源但动态恢复能力不足。

10. 这一部分的目标是把 benchmark instance 和 evaluation protocol 统一起来：场景状态 \(S\) 定义模型面对的救援世界，决策链条 \(O \rightarrow F \rightarrow R \rightarrow P \rightarrow P'\) 定义模型必须完成的闭环能力，而六个指标用于解释模型在任务完成、响应时效、任务耗时、事件稳定性、资源压力和多 UAV 协同方面的表现。
